# MaiSaka 好友申请处理插件 (Friend Request Handler)

> QQ 好友申请自动化处理插件。它可以将接收到的新好友申请推送到指定的管理员 QQ，并允许管理员直接在私聊中快捷同意或拒绝。
> 有问题请提issue

## ✨ 主要功能

* **Webhook 监听**：在本地启动一个轻量级 HTTP 服务器，专门接收 NapCat 推送的 `friend` request 事件上报。
* **详细资料推送**：收到申请后，自动抓取申请人的详细 QQ 资料（昵称、性别、年龄、等级、所在地、个性签名、验证消息等），并私聊推送给所有配置的管理员。
* **快捷指令审批**：管理员可以通过简单的私聊指令（`/同意` 或 `/拒绝`）快速处理申请。
* **自动欢迎与备注**：通过好友申请后，支持自动发送自定义的欢迎语，并可自动设置好友备注。
* **启动补漏机制**：插件启动时会自动轮询一次 NapCat 的系统消息，防止在插件未运行或重启期间漏掉好友申请。

## ⚙️ 配置说明

插件首次加载后，会在 MaiBot 的配置面板中生成对应的设置项。

### 1. MaiBot 插件端配置

在机器人的配置界面（或配置文件）中，你可以按模块设置以下内容：

* **管理员设置** (`admin`)
* `admin_qqs`: 管理员 QQ 号列表。只有这里的 QQ 能接收到申请推送并使用审批指令。


* **Webhook 设置** (`webhook`)
* `host`: 监听地址，默认 `127.0.0.1`。
* `port`: 监听端口，默认 `18080`。
* `path`: 监听路径，默认 `/maibot/friend_request`。
* `secret`: 签名密钥（可选）。


* **启动补漏** (`startup_sweep`)
* `enabled`: 是否启用启动补漏。
* `delay_sec`: 启动后延迟几秒执行（留给适配器连接时间），默认 `10`。
* `fetch_count`: 单次拉取的最大数量，默认 `20`。


* **欢迎语** (`welcome`)
* `message`: 同意申请后自动发送的欢迎语。
* `remark`: 通过时自动设置的好友备注（留空则不设置）。



### 2. NapCat 端配置

为了让插件能收到好友申请，你需要前往 NapCat 的配置界面，添加一个 **HTTP 客户端 (HTTP Post)** 推送目标：

* **URL**: `http://127.0.0.1:18080/maibot/friend_request` （请根据你在插件中配置的 host, port, path 进行替换）。
* **Token / Secret**: 如果你在插件的 `webhook.secret` 中填写了密钥，请在这里填入相同的值；如果插件端留空，则此处也留空。

## 🚀 使用方法

当有用户添加机器人为好友时，管理员 QQ 会收到类似如下的推送消息：

> 📩 收到新的好友申请
> QQ号: 123456789
> 昵称: 张三
> 性别: 男
> 年龄: 22
> 验证消息: 你好，我想用机器人
> 通过申请请发送：/同意 123456789
> 拒绝申请请发送：/拒绝 123456789

**管理员审批指令：**

| 指令 | 说明 | 示例 |
| --- | --- | --- |
| `/同意 <QQ号>` | 同意该用户的好友申请，并发送欢迎语 | `/同意 123456789` |
| `/拒绝 <QQ号>` | 拒绝该用户的好友申请 | `/拒绝 123456789` |

> **注意：** 审批指令需要管理员和机器人处于**私聊状态**下发送，且发送者的 QQ 必须在 `admin_qqs` 列表中。

## 📁 目录结构与数据存储

插件会在自身目录下自动生成一个 `data` 文件夹用于持久化数据：

* `data/state.json`: 用于记录**待处理的申请列表**和**已推送的事件 Flag**。这保证了即使机器人重启，之前的申请依然可以用指令处理，并且启动补漏不会重复推送已经通知过的申请。无需人工干预此文件。
